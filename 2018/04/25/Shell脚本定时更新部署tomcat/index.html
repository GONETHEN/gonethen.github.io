Shell脚本定时更新部署tomcat

发表于 2018/04/25 | 分类于 技术 | 阅读量 次

原来打算用Python3，结果发现Shell更简单。

前言

​ 终于实现了一个简单的自动化部署，喜极而泣。顺便学习了一下Shell语言，嗯，学无止境。

需求

  1. 为了实现服务器自动化部署

实现步骤

1. 编写Shell脚本

  1. 在tomcat目录下执行mkdir autodeploy新建文件夹
  2. 进入autodeploy目录，执行touch autodeploy.sh新建脚本文件

2. 准备要更新的文件

  1. 将项目类文件目录com打包成zip
  2. 将zip文件上传到autodeploy目录下

3. 定时执行Shell脚本

  1. 需要用到crond服务，有些Linux已经安装，如果没有安装执行下面两句进行安装：

    yum install vixie-cron

    yum install crontabs

    crond服务的相关操作：

    service crond start //启动服务

    service crond stop //关闭服务

    service crond restart //重启服务

    service crond reload //重新载入配置

    service crond status //查看crontab服务状态

    chkconfig –level 35 crond on //加入开机自动启动

  2. 执行yum install -y ntsysv安装ntsysv，并执行ntsysv查看crond.service服务是否已设置为开机启动

  3. 新增调度任务有两种方法，第一种我尝试没有成功所以选择了第二种

    • 执行vi /etc/crontab编辑/etc/crontab文件，添加相应的任务

      默认文件格式：

      顾名思义：

      星号（）：代表所有可能的值，例如month字段如果是星号，则表示在满足其它字段的制约条件后每月都执行该命令操作
      逗号（,）：可以用逗号隔开的值指定一个列表范围如，“1,2,5,7,8,9”
      中杠（-）：可以用整数之间的中杠表示一个整数范围，例如“2-6”表示“2,3,4,5,6”
      正斜线（/）：可以用正斜线指定时间的间隔频率，例如“0-23/2”表示每两小时执行一次。同时正斜线可以和星号一起使用，例如      /10，如果用在minute字段，表示每十分钟执行一次

    • 执行crontab -e，然后添加相应的任务

      添加的内容跟编辑/etc/crontab文件一样，我想要我的Shell脚本在2018年4月26日4:30执行，那么就在第一行写入：

      30 4 26 4 * /storage/lhTomcat/autodeploy/autodeploy.sh /storage/lhTomcat/autodeploy/com

      但是crond是周期性执行的，所以我们在使用完之后需要删除这个调度任务

  4. 执行crontab -l -u root查看用户root的所有调度任务

  5. ​

  6. 任务完成后记得执行crontab -r删除所有调度任务
