Mysql查询技巧

发表于 2018-04-08 | 分类于 技术 | | 阅读量 次

学无止境。

前言

​ 前段时间公司要月度报表数据，由于没有专业的数据分析人员，这份工作自然而然的就落到了我的身上。之前多少也接触过Mysql，left join更是自认为用得很溜，结果这么一个sql还是写了一天，结果倒是出来了，具体对不对就让时间来考验吧。

需求

  1. 这是一个月度报表
  2. 用到的表可分为三大类：用户表account/代理表manager/其他业务相关表order等
  3. 查询结果应为：以每个总代理为一条记录，查询所属于他的所有子代理的月度流水和所属于他或他的子代理的所有用户的月度流水

第一版

​ 由于二月份代理的业务数据一只手就数的过来，所以二月份做的第一版只涉及到用户月度流水查询，代理的数据直接计算器算出来的。

小结

​ 这一版其实我也学到了不少东西，比如：

  • 如果字段值为空则返回指定值IFNULL(c.number,0)/字符串转时间格式STR_TO_DATE('2018-03-01 00:00:00',"%Y-%m-%d %T")
  • manager和account是一对多，account和carddeductlog/techargeMoney/playerorder也是一对多，这条语句的思路就是：
    • 根据subCode字段左连接查出每个总代理下的所有子代理，再根据manager_up_id左连接查出每个用户的每个代理
    • 查出每个业务表当月的数据，再以外键accountid做分组统计得到每个account当月的数据
    • 用第一步的数据左连接第二部的数据，最后求和并GROUP BY m.id得到最终结果
